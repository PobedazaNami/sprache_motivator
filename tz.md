Название: Attention & Value Pack для аудитории A2-B1

Роль агента:
Ты работаешь в существующем проекте Sprache Motivator. Твоя задача не добавить "еще функций", а сделать продукт более цепляющим, полезным и понятным для пользователей уровня A2-B1.

Цель релиза:
Увеличить внимание пользователя, время внутри продукта и ощущение пользы от обучения.
После изменений пользователь должен понимать:
- что делать прямо сейчас
- зачем ему это действие
- какую практическую пользу он получил
- какой у него следующий шаг

Контекст:
Сейчас продукт уже имеет Daily Trainer, Express Trainer, Flashcards Mini App, Subtitle Trainer, progress, streak, reports, friends stats.
Проблема в том, что это ощущается как набор отдельных режимов, а не как единый маршрут обучения.

Основной продуктовый принцип:
- меньше абстрактной геймификации
- больше коротких сценариев
- больше полезных жизненных фраз
- меньше перегруза теорией
- фокус на A2-B1, а не на абстрактный B2-формат

Что нужно реализовать:

1. Новый основной вход: "Маршрут на сегодня"
- Добавить в главное меню новую заметную кнопку:
  - uk: `🔥 Сьогоднішній маршрут`
  - ru: `🔥 Маршрут на сегодня`
- По нажатию показывать короткий daily route из 3 шагов:
  - 1 задание Express Trainer
  - 1 переход в повторение карточек, если есть due cards
  - 1 переход в Subtitle Trainer или fallback-задачу
- Если шаг недоступен, показывать fallback:
  - нет карточек -> CTA создать первые 3 карточки
  - нет due cards -> CTA добавить слова в карточки
  - видео недоступно -> CTA на Express Trainer
- Пользователь не должен выбирать обучение с нуля между 5-6 режимами. Продукт должен сам предложить первый маршрут.

2. Сценарное обучение для A2-B1
- Усилить подачу заданий в Daily Trainer и Express Trainer.
- Каждое задание должно иметь короткий жизненный label сценария:
  - в магазине
  - у врача
  - на работе
  - знакомство / small talk
  - в переписке
  - в транспорте
  - аренда жилья / соседи
  - документы / бюрократия
  - звонок / запись / просьба
  - бытовая проблема
- Не ломать текущую topic-based систему.
- Поверх текущих TrainerTopic добавить более "человеческую" упаковку в UI и генерации контента.
- Генерация предложений должна стать:
  - короче
  - полезнее
  - ближе к реальной речи
  - проще для A2-B1

3. Улучшить фидбек после ответа
- После каждого ответа показывать не только correction/explanation, но и практическую пользу.
- Новая структура ответа:
  - оценка
  - что исправить
  - как сказать правильно
  - готовая фраза для жизни
- Для правильных ответов показывать не просто похвалу, а что именно пользователь теперь умеет сказать.
- Для неправильных ответов объяснение должно быть коротким, практичным и без перегруза.

4. Progress 2.0
- Переработать экран "Мой прогресс", чтобы он отвечал на вопрос "что делать дальше".
- Добавить:
  - прогресс по сегодняшнему маршруту
  - количество due flashcards
  - количество освоенных полезных фраз / предложений
  - 1 рекомендованный следующий шаг
- Если возможно без тяжелой архитектуры, показать:
  - 1 сильную тему
  - 1 слабую тему
- Формат рекомендации:
  - "Сейчас лучше всего: пройти 5 карточек"
  - "Следующий шаг: 1 экспресс-задание"
  - "Чтобы сохранить серию, выполните 1 короткое задание сегодня"

5. Возврат и мягкая мотивация
- Реализовать comeback flow после 2+ дней без активности.
- Использовать уже существующие механизмы inactivity/streak, если они есть.
- Добавить короткие comeback-сообщения без давления:
  - "Вернемся с короткой сессии на 2 минуты"
  - "Сегодня можно просто сделать 1 полезную фразу"
  - "Чтобы вернуть ритм, начните с экспресс-тренажера"
- Добавить daily tip в подходящие точки входа, если это можно сделать без перегруза UX.

Обязательные продуктовые исправления по пути:
- Subtitle Trainer не должен молча использовать `uk` как target language для пользователей `ru`.
- Flashcards dashboard должен корректно различать `new`, `due`, `learning`, `known`.
- Если hint activation обещается в тексте, он должен реально трекаться, либо текст нужно скорректировать.
- Если затрагивается scheduler/user timezone, нельзя оставлять жесткую привязку UX только к `Europe/Kyiv`, если модель уже предполагает user timezone.

Ограничения:
- не перестраивать админку
- не делать сложную систему бейджей/лиг/валют
- не делать голосовой модуль
- не делать тяжелые миграции, которые могут поломать текущие данные
- не ломать текущие flows: trainer, express, flashcards, subtitle trainer, settings, friends

Критерии приемки:
- В главном меню есть новый понятный вход в обучение через daily route.
- Пользователь может пройти короткий маршрут дня без самостоятельного выбора всего flow.
- Daily и Express Trainer подаются как жизненные сценарии для A2-B1.
- После ответа пользователь видит практическую пользу, а не только correction.
- Экран прогресса дает следующий рекомендуемый шаг.
- Есть мягкий comeback flow после неактивности.
- Все тексты локализованы для uk и ru.
- Текущая функциональность не сломана.

5. Обновить progress screen.
